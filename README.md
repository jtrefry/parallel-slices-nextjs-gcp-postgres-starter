# Next.js GCP PostgreSQL application

This starter project was generated by
[Parallel Slices](https://github.com/jtrefry/parallel-slices). It combines a
Next.js Turborepo and Mantine UI baseline with PostgreSQL, Google Cloud delivery
contracts, AI planning workflows, isolated slice workers, quality gates, review
evidence, and safe Git controls.

- Default controller: `Cursor`
- Architecture: `nextjs-gcp-postgres`
- Package manager: `pnpm`

> The Parallel Slices bootstrap has already run. Do not run `create-turbo` again
> in this repository.

## Prerequisites

Before initialization or use after a fresh clone, provide:

- macOS or Linux, Git, and Node.js 22 LTS or 24 LTS;
- the package manager pinned by the root `package.json`;
- Cursor, Codex, or Claude Code installed and signed in;
- at least one configured independent review provider CLI—Codex, Claude Code,
  or Antigravity—installed and signed in before its first review;
- Docker Desktop for the later container-backed integration and E2E foundation;
  Rancher Desktop with `dockerd (moby)` is a free, best-effort alternative;
- GitHub CLI only when initialization is authorized to create or publish a
  GitHub repository;
- the exact Trivy version in `.trivy-version` before local full-pipeline runs;
  and
- supported Google Cloud emulators only when this product's initialized tests
  require them.

Review the upstream [model-usage expectations and prerequisites](https://github.com/jtrefry/parallel-slices/blob/main/README.md#plan-for-model-usage)
before selecting controller and reviewer subscriptions, quotas, or API budgets.

## Getting started

Run these steps from the repository root:

1. Install the locked dependencies:

   ```bash
   node scripts/parallel-slices/corepack-runner.mjs pnpm install --frozen-lockfile
   ```

2. Choose a controller and open its version-matched local guide:
   [Codex](docs/parallel-slices/using-codex.md),
   [Cursor](docs/parallel-slices/using-cursor.md), or
   [Claude Code](docs/parallel-slices/using-claude-code.md).
3. Run that guide's initialization command.
4. Describe the product, answer the consequential questions, and review the
   generated Product Plan.
5. Approve the Product Plan when it accurately captures the product. AI then
   commits it, compiles the optimized execution map separately, and obtains
   independent AI planning approval before implementation can begin.
6. Use the same controller guide's preparation command to review and start the
   exact `/loop` or `/goal` invocation.

For every later feature or fix, use that controller's planning and preparation
commands again. Do not bypass the Product Plan approval or compiled slice map
with an unbounded implementation prompt.

## Local development before initialization

The generated baseline exposes these checks immediately:

```bash
node scripts/parallel-slices/corepack-runner.mjs pnpm run lint
node scripts/parallel-slices/corepack-runner.mjs pnpm run check-types
node scripts/parallel-slices/corepack-runner.mjs pnpm run build
```

Start both Next.js applications:

```bash
node scripts/parallel-slices/corepack-runner.mjs pnpm run dev
```

Initialization plans the complete project-specific quality foundation described
by `.parallel-slices/config.json`. Foundation slices then implement the missing
format, SQL security, unit, integration, E2E, and Trivy scripts and prove them
before product slices may run.

## Repository layout

- `apps/docs/` is a starter Next.js application with the workspace package
  name `docs`.
- `apps/web/` is the other starter Next.js application.
- `docs/` is the root for repository documentation. It is separate from
  `apps/docs/`.
- `docs/project/` holds the unique application-specific contracts created
  during initialization.
- `docs/plans/` holds human-approved Product Plans, compiled scope manifests,
  durable run state, and permanent review evidence.
- `docs/parallel-slices/` holds the version-matched procedures used by this
  checkout's agents and runtime.

All paths are relative to the repository root. The Next.js application at
`apps/docs/` is not the documentation directory at `docs/`.

## Ownership and safety

`.parallel-slices/architecture.json` records the immutable installed
architecture package, version, manifest hash, resolved options, files, and
quality floors. Changing it requires an explicit architecture migration.

All three native controller adapters are installed. The default controller is
a convenience, not exclusive ownership. Each sliced run records its actual
controller in durable state and uses an ignored local lease to prevent another
tool from taking over concurrently.

The repository starts in local-only publication mode. Initialization must
explicitly record an authorized GitHub repository before the controller may
publish one goal branch and pull request. Nothing in this README authorizes
provider review, push, merge, deployment, cloud mutation, package publication,
release creation, or production migration.
